#### 1. ⭐⭐⭐⭐⭐ 基于MQTT的报警系统

**功能描述**：
- 订阅MQTT报警主题接收实时报警
- 报警事件触发和处理
- 报警确认流程
- 报警统计分析
- 报警推送通知

**为什么重要**：
- 实时监控生产安全的关键功能
- 通过MQTT订阅实现低延迟报警响应
- 事件驱动架构,减少轮询开销
- 支持分布式报警源

**技术方案**：
```typescript
export interface MqttAlarmSystem {
  // MQTT报警订阅配置
  alarmTopicConfig: {
    baseTopic: 'alarm/',           // 基础主题
    topics: {
      critical: 'alarm/critical',  // 严重报警
      high: 'alarm/high',          // 高级报警
      medium: 'alarm/medium',      // 中级报警
      low: 'alarm/low'             // 低级报警
    }
  }

  // 报警消息格式 (MQTT Payload)
  alarmMessage: {
    id: string              // 报警唯一ID
    timestamp: number       // 时间戳
    level: 'critical' | 'high' | 'medium' | 'low'
    type: 'threshold' | 'deviation' | 'statusChange' | 'communication'
    deviceId: string        // 设备ID
    tagId: string          // 测点ID
    message: string        // 报警描述
    value?: number         // 当前值
    threshold?: number     // 阈值
    acknowledged: boolean  // 是否已确认
  }

  // 订阅报警主题
  subscribeAlarmTopics(): void

  // 报警消息处理
  handleAlarmMessage(topic: string, payload: AlarmMessage): void

  // 报警确认（发布到MQTT）
  acknowledgeAlarm(alarmId: string, operator: string, comment: string): void

  // 报警统计
  getAlarmStatistics(): AlarmStatistics
}
```

**核心功能**：

1. **MQTT主题订阅**
   ```typescript
   // 订阅所有报警主题
   fuxaMqttService.subscribe('alarm/#', 1)
   
   // 订阅特定级别报警
   fuxaMqttService.subscribe('alarm/critical', 2)
   fuxaMqttService.subscribe('alarm/high', 1)
   ```

2. **报警事件处理**
   ```typescript
   // 监听MQTT消息
   fuxaMqttService.on('message', (topic, payload) => {
     if (topic.startsWith('alarm/')) {
       const alarm = JSON.parse(payload.toString())
       handleAlarmEvent(alarm)
     }
   })
   
   // 处理报警事件
   function handleAlarmEvent(alarm: AlarmMessage) {
     // 1. 显示报警弹窗
     showAlarmPopup(alarm)
     
     // 2. 播放报警声音
     playAlarmSound(alarm.level)
     
     // 3. 触发组件事件
     triggerComponentEvent(alarm)
     
     // 4. 保存到本地缓存
     saveAlarmToCache(alarm)
   }
   ```

3. **报警触发组件事件**
   ```typescript
   // 报警时自动触发组件动作
   function triggerComponentEvent(alarm: AlarmMessage) {
     // 查找绑定该设备的组件
     const components = findComponentsByDevice(alarm.deviceId)
     
     components.forEach(component => {
       // 触发闪烁动画
       if (alarm.level === 'critical') {
         component.animation = 'blink'
       }
       
       // 改变颜色
       component.fillColor = getAlarmColor(alarm.level)
       
       // 跳转到相关画面
       if (component.events?.alarm) {
         executeAction(component.events.alarm.actions)
       }
     })
   }
   ```

**MQTT主题结构设计**：
```
alarm/
├── critical/          # 严重报警主题
│   ├── {deviceId}/   # 按设备分组
│   └── {tagId}       # 按测点分组
├── high/             # 高级报警主题
├── medium/           # 中级报警主题
├── low/              # 低级报警主题
├── ack/              # 报警确认主题
│   └── {alarmId}     # 确认特定报警
└── clear/            # 报警清除主题
    └── {alarmId}     # 清除特定报警
```

## 💡 性能优化建议

### 1. 虚拟滚动
对于大量组件的画面，使用虚拟滚动只渲染可见区域

```typescript
import { useVirtualList } from '@vueuse/core'

const { list, containerProps, wrapperProps } = useVirtualList(
  components,
  { itemHeight: 100, overscan: 5 }
)
```

### 2. Web Worker
将数据处理放到 Worker 线程，避免阻塞 UI

```typescript
// data-worker.ts
self.onmessage = (e) => {
  const data = e.data
  const processed = processData(data)
  self.postMessage(processed)
}
```

### 3. 防抖节流
```typescript
import { throttle, debounce } from 'lodash-es'

// 高频数据更新使用节流
const throttledUpdate = throttle((data) => {
  updateComponent(data)
}, 100) // 100ms

// 搜索框使用防抖
const debouncedSearch = debounce((keyword) => {
  searchComponents(keyword)
}, 300) // 300ms
```

### 4. 增量更新
只更新变化的数据点，不要全量刷新

```typescript
// ✅ 好的做法
function updateChanged(changedIds: string[]) {
  changedIds.forEach(id => {
    const component = findComponent(id)
    updateComponent(component)
  })
}
```

### 5. 组件懒加载

## 📝 关于图层管理的说明

**图层管理功能评估**：

图层管理在以下场景下**非常有用**：
- ✅ 画面组件超过30个时，需要分层组织
- ✅ 有多个设计师协作编辑同一画面
- ✅ 需要快速显示/隐藏一组组件（如背景装饰）
- ✅ 需要锁定背景层防止误操作

但如果你的使用场景是：
- ❌ 每个画面组件数量较少（<20个）
- ❌ 单人编辑，不需要复杂的图层管理
- ❌ 画面结构简单，没有复杂的层次关系

那么图层管理可以**暂不实现**，等后续有需求再添加。

**建议**：先实现核心功能（报警、多画面、撤销重做），如果在实际使用中发现画面管理困难，再考虑添加图层功能。
