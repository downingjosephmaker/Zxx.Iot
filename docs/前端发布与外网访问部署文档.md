# Zxx.Iot 前端发布与外网访问部署文档

> 目标：把 `vuefrontend` 前端发布出来，并让**外网临时访问本机前端**。
> 已落地方案：**WSL(openEuler) nginx 托管前端 + Windows cloudflared 命名隧道暴露公网**。
> 公网入口：`https://iot.fhhzxx.site`

---

## 一、架构总览

```
外网用户浏览器
   │ https://iot.fhhzxx.site
   ▼
Cloudflare 边缘节点（自动 HTTPS 终结）
   │ 加密隧道（cloudflared 出站长连，无需公网IP/开端口）
   ▼
Windows 本机 cloudflared 服务(cloudflaredservice)
   │ ingress: iot.fhhzxx.site → http://localhost:8080
   ▼
WSL(openEuler) nginx :8080
   ├─ /            → 前端 dist (/home/nginx/html/iot)
   ├─ /Api         → 反代 http://127.0.0.1:13699 (IotWebApi 生产)
   └─ /signalr     → 反代 http://127.0.0.1:13699 (WebSocket 长连)
   ▼
Windows 本机 IotWebApi :13699 (生产模式)
```

**职责划分**：
- **nginx** 负责“发布”——把前端 `dist` 变成网站、反代后端 API/SignalR。
- **Cloudflare Tunnel** 只负责“让外网进得来”——不用公网 IP、不用开放端口、自动套 HTTPS。
- 两者不是二选一，而是黄金搭档。

## 二、关键环境事实（本机实测）

| 项目 | 实际值 |
|------|--------|
| WSL 发行版 | openEuler-25.09 |
| nginx 部署 | docker-compose，`network_mode: host` |
| nginx 真实配置目录 | `/home/nginx/`（**不是** `/home/docker_yml/nginx/nginx/`，那是模板副本） |
| nginx 站点目录 | `/home/nginx/conf.d/`（已有 default.conf:8000、8668.conf、8669.conf） |
| Iot 前端目录 | `/home/nginx/html/iot/`（本方案新增） |
| Iot nginx 端口 | **8080**（8000 被 default 占用，8081 被 docker 占用，故选 8080） |
| IotWebApi 生产端口 | **13699**（`appsettings.json` 的 `Urls: http://*:13699`） |
| IotWebApi 开发端口 | 15088（`appsettings.Development.json`，本方案不用） |
| SignalR Hub 路径 | `/signalr/chatHub`（`Program.cs` MapHub，注意 H 大写） |
| cloudflared | Windows 版 2026.5.2，`C:\Program Files (x86)\cloudflared\` |
| 隧道 ID | `f6f6ee9d-4226-43cd-b2d2-0b334a9d38d8`（与 AiNet 共用，一隧道多主机名） |
| 认证目录 | `C:\Users\Administrator\.cloudflared\`（cert.pem + 隧道凭证 json + config.yml） |
| 域名 | `fhhzxx.site`（已托管在 Cloudflare） |

> **前端“运行时可配后端地址”银弹**：`src/config/index.ts` 的 `getApiUrl()` 优先读 `public/platform-config.json` 的 `ApiUrl`，其次回退打包 `.env`。因此**打一次包即可，切换后端只改 dist 里的 json，无需重打包**。本方案把 `ApiUrl` 设为相对路径 `/Api`，前端跟随访问域名自适应（本地/隧道通用）。

## 三、已落地的部署步骤（复现用）

### 步骤 1：前端构建
```bash
cd D:/MyWork/Project/My/Zxx.Iot/vuefrontend
pnpm build          # 产出 dist（hash 路由模式）
```

### 步骤 2：改 dist 产物的运行时配置（不动源码）
编辑 `vuefrontend/dist/platform-config.json`，把三处设为相对路径：
```json
"ApiUrl": "/Api",
"WapianUrl": "/htmlstatic/",
"SignalRUrl": "/signalr/chatHub"
```

### 步骤 3：拷贝 dist 到 WSL nginx
在 WSL(openEuler) 内执行（WSL 通过 /mnt/d 访问 Windows 盘）：
```bash
SRC=/mnt/d/MyWork/Project/My/Zxx.Iot/vuefrontend/dist
DST=/home/nginx/html/iot
mkdir -p "$DST" && rm -rf "$DST"/* && cp -r "$SRC"/. "$DST"/
```

### 步骤 4：写 nginx 站点配置
新建 `/home/nginx/conf.d/iot.conf`（完整内容见附录 A），然后在 nginx 容器内校验并热重载：
```bash
docker exec nginx nginx -t          # 校验语法
docker exec nginx nginx -s reload   # 热重载（不中断现有连接）
```
本地自测：`curl http://localhost:8080/` 应返回 HTTP 200。`/Api` 返回 502 是正常的——只要后端没起，通道本身是通的。

### 步骤 5：cloudflared 加 Iot ingress
编辑 `C:\Users\Administrator\.cloudflared\config.yml`，在 ingress 里加一条（**带 hostname 的规则必须在 http_status:404 兜底之前**）：
```yaml
tunnel: f6f6ee9d-4226-43cd-b2d2-0b334a9d38d8
credentials-file: C:\Users\Administrator\.cloudflared\f6f6ee9d-4226-43cd-b2d2-0b334a9d38d8.json

ingress:
  - hostname: ainet-media.fhhzxx.site
    service: http://localhost:15800
  - hostname: iot.fhhzxx.site          # ← 新增
    service: http://localhost:8080      # ← 指向 nginx
  - service: http_status:404
```
校验：
```powershell
& "C:\Program Files (x86)\cloudflared\cloudflared.exe" --config "C:\Users\Administrator\.cloudflared\config.yml" tunnel ingress validate
```

### 步骤 6：DNS 路由 + 重启服务
```powershell
# 创建 iot 子域 CNAME 指向隧道
& "C:\Program Files (x86)\cloudflared\cloudflared.exe" tunnel --config "C:\Users\Administrator\.cloudflared\config.yml" route dns f6f6ee9d-4226-43cd-b2d2-0b334a9d38d8 iot.fhhzxx.site

# 重启服务让新 ingress 生效（会让共用隧道的 AiNet 媒体瞬断 2-5 秒自恢复）
Restart-Service cloudflaredservice -Force
```

### 步骤 7：验证
```powershell
Invoke-WebRequest -Uri 'https://iot.fhhzxx.site/' -UseBasicParsing   # 应 200，含 <div id="app">
```

## 四、日常更新前端（改代码后）

只需重复步骤 1 → 2 → 3，然后**无需动 nginx/cloudflared**：
```bash
pnpm build
# 改 dist/platform-config.json 的 ApiUrl=/Api（若脚本化可跳过）
# 在 WSL: rm -rf /home/nginx/html/iot/* && cp -r /mnt/d/.../dist/. /home/nginx/html/iot/
```
浏览器强刷即可（静态资源带 hash，天然破缓存）。

## 六、故障排查

| 现象 | 原因 | 处理 |
|------|------|------|
| 公网 502 | 后端 IotWebApi 未在 13699 运行 | 启动后端（生产模式，监听 13699） |
| 公网 404（cloudflare 页） | ingress 未匹配 / 服务没重启 | 检查 config.yml，`Restart-Service cloudflaredservice` |
| 前端白屏、接口跨域 | platform-config.json 的 ApiUrl 没设成 /Api | 改 dist 里的 json 为相对路径 |
| SignalR 连不上 | hub 路径大小写不符 | 确认前端 `/signalr/chatHub`（H 大写）与后端 MapHub 一致 |
| 隧道离线 | cloudflaredservice 停了 | `Start-Service cloudflaredservice`；`cloudflared tunnel info <ID>` 看连接器 |

## 附录 A：`/home/nginx/conf.d/iot.conf` 完整内容

```nginx
server {
    listen       8080;
    listen  [::]:8080;
    server_name  localhost iot.fhhzxx.site;

    location / {
        root   /home/nginx/html/iot;
        index  index.html index.htm;
        try_files $uri $uri/ /index.html;
    }

    location /Api {
        add_header 'Access-Control-Allow-Origin' *;
        add_header 'Access-Control-Allow-Credentials' 'true';
        add_header 'Access-Control-Allow-Methods' 'GET, POST, PUT, DELETE';
        add_header 'Access-Control-Allow-Headers' '*';
        if ($request_method = 'OPTIONS') { return 204; }
        proxy_pass http://127.0.0.1:13699;
        proxy_set_header Host $http_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    location /signalr {
        add_header 'Access-Control-Allow-Origin' *;
        add_header 'Access-Control-Allow-Credentials' 'true';
        add_header 'Access-Control-Allow-Methods' 'GET, POST, PUT, DELETE';
        add_header 'Access-Control-Allow-Headers' '*';
        if ($request_method = 'OPTIONS') { return 204; }
        proxy_pass http://127.0.0.1:13699;
        proxy_set_header Host $http_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 3600s;
    }
}
```

## 附录 B：Windows 纯本机备选方案（不依赖 WSL nginx）

若不想用 WSL 里的 nginx，可在 **Windows 本机**用一份独立 nginx 托管前端，cloudflared 直接指向它。适合把整套发布收敛到 Windows 一侧。

### B.1 准备 Windows nginx
1. 下载 nginx for Windows（http://nginx.org/en/download.html），解压到如 `C:\nginx`。
2. 把前端 `dist` 拷到 `C:\nginx\html\iot\`。
3. 改 dist 里的 `platform-config.json` 的 `ApiUrl` = `/Api`（同 WSL 方案）。

### B.2 Windows nginx 站点配置
在 `C:\nginx\conf\conf.d\iot.conf`（或直接写进 `nginx.conf` 的 http 块），端口选一个 Windows 上空闲的，如 **8082**：
```nginx
server {
    listen       8082;
    server_name  localhost iot.fhhzxx.site;

    location / {
        root   C:/nginx/html/iot;
        index  index.html;
        try_files $uri $uri/ /index.html;
    }

    location /Api {
        if ($request_method = 'OPTIONS') { return 204; }
        proxy_pass http://127.0.0.1:13699;
        proxy_set_header Host $http_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /signalr {
        proxy_pass http://127.0.0.1:13699;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $http_host;
        proxy_read_timeout 3600s;
    }
}
```
> Windows nginx 直连 127.0.0.1:13699 更简单——后端和 nginx 同在 Windows，无跨 WSL 问题。

启动：
```powershell
cd C:\nginx
start nginx           # 启动
nginx -s reload       # 改配置后热重载
nginx -s stop         # 停止
```

### B.3 cloudflared 指向 Windows nginx
把 config.yml 里 Iot 那条 service 改成 Windows nginx 端口：
```yaml
  - hostname: iot.fhhzxx.site
    service: http://localhost:8082      # ← 指向 Windows nginx
```
其余（route dns、Restart-Service）同主方案步骤 6。

### B.4 极简临时方案（无域名时）
若只是临时演示、连域名都不想配，一条命令给随机公网地址（quick tunnel）：
```powershell
& "C:\Program Files (x86)\cloudflared\cloudflared.exe" tunnel --url http://localhost:8082
# 输出里会给一个 https://xxxx.trycloudflare.com 地址，进程关掉即失效
```

| 维度 | WSL nginx（已落地） | Windows nginx（备选） | quick tunnel（临时） |
