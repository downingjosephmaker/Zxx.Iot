## 3. 执行剧本(停服窗口内照抄,每步可回滚)

> **铁律:DDL 必须在 CodeFirst 触发之前跑。启动后端 = CodeFirst 抢跑窗口打开。**

### 步骤 1 · 备份(硬门槛,不可省)
```bash
# 在能连到 6305/zhjngkdb 的机器上执行,产出实际 dump 到 database/backup/
pg_dump -h 127.0.0.1 -p 6305 -U <user> -d zhjngkdb -F c \
  -f database/backup/zhjngkdb_pre_p3migration_20260710.dump
# 校验 dump 大小非 0
```

### 步骤 2 · 停服
```
停掉 Iot 后端(生产端口 13699 / 本地 Development 口)。
确认进程已退出,无残留连接会触发 CodeFirst。
```

### 步骤 3 · 现场核对(执行前 3 查)
```sql
-- (a) 查实际含 unit_id 的业务表(应命中 V0001 fixed_tables 里的表)
SELECT table_name FROM information_schema.columns
 WHERE table_schema = current_schema() AND column_name = 'unit_id'
 ORDER BY table_name;

-- (b) 枚举 event_history/run/signal 的所有已存在分表实例
SELECT table_name FROM information_schema.tables
 WHERE table_schema = current_schema()
   AND (table_name ~ '^event_history_\d+$'
     OR table_name ~ '^event_run_\d+$'
     OR table_name ~ '^event_signal_\d+$')
 ORDER BY table_name;

-- (c) 确认 basicunit_info.unit_id 仍在(不应被改)
SELECT column_name FROM information_schema.columns
 WHERE table_name = 'basicunit_info' AND column_name IN ('unit_id','tenant_id');
```

### 步骤 4 · 跑正向 DDL(unit_id → tenant_id)
```
psql -h 127.0.0.1 -p 6305 -U <user> -d zhjngkdb \
  -f database/business/V0001__rename_unit_to_tenant.sql
# 观察 RAISE NOTICE 输出:每张表应打印 [改名] / [跳过] / [无列];
# 分表实例应逐个打印 [改名·分表]。
```
> 脚本幂等,可重复跑。若 CodeFirst 曾误加空 tenant_id,脚本会自动"DROP 空列再 RENAME"。

### 步骤 5 · D 类 hypertable(现场决策)
```sql
-- 先查 iot_ts.device_event_log 是否存在
SELECT 1 FROM information_schema.tables
 WHERE table_name = 'device_event_log';
```
- **不存在**(遥测未启用) → 跳过,同步把 `V0005__event_logs.sql:17` 源脚本的 `unit_id` 改为 `tenant_id`(供未来建表用)。
- **存在** → 现场测 `ALTER TABLE ... RENAME COLUMN unit_id TO tenant_id` 在 TimescaleDB(尤其已压缩 hypertable)的兼容性;通过则改,并同步 V0005 源脚本;不通过则记录待专项处理。

### 步骤 6 · (可选)租户树 P3-B
> 决策 C1 定"同窗口一起做"。若本次只做物理迁移,可拆到下一批。若一起做:
