## 一、已拍板决策

| # | 决策 | 结论 |

## 二、现状事实（探查结论，非推测）

**引擎侧**

- 画布数据存 `ScadaProjectData.ContentData`（整个项目 JSON 序列化），结构 `{ settings, components, devices, datasets }`（`main/utils-project.ts:538`）
- 编辑器入口 `/scada/editor/:id`（`router/modules/remaining.ts:46`），项目 ID 走 **路由 params**
- 编辑器渲染核心 `createComponentElement`（`main/utils1.ts:634`），21 个参数，但各 `create*Element` 分支均可独立 import；对编辑器的真实耦合只有 `editorContainer`（需含 `.canvas-content`）与 `setupComponentInteractions`

**⚠️ 运行态是空壳（本方案最大工程量）**

- `runtime.vue:166-181` 的 `renderComponent` 只处理 `text`/`image`/`video`，其余组件（200+ SVG 图元、`unified-chart`、`table`、`text-card`）**全部渲染为空 div**
- 数据绑定 `applyDatasetValues`（`runtime.vue:141-161`）只写 `properties.text`，仅对文本组件有效
- 路由（`/scada/runtime/:id`）、项目加载、SignalR + MQTT 订阅链路**都是通的**，唯独渲染层没写

**载体侧**

- `DashProject` / `DashProjectData` 与 `ScadaProject` / `ScadaProjectData` **字段完全同构**，DAO + Controller 全在，已实现 `ITenantEntity`，前端零引用 → 可直接承载报表项目
- `ScadaProject` 无 `ProjectType` 字段 —— 但按 D2 走双表，**不需要加**

**报表组件现状**

| 需求 | 现有组件 | 分类 |
|---|---|---|
| 表格 | `table` | controls |
| 图表 | `unified-chart`（走 `utils4.ts:114 initEChart`） | controls |
| 文本卡片 | `text-card` | basic |
| 统计数值卡 | **缺，需新造**（最接近的是 `led-display`） | — |

报表组件散落在 `basic`/`controls` 两类中，按分类过滤会连带拖入按钮/滑块/开关 → **过滤粒度必须下沉到组件 name 白名单**。

**导出依赖：零新增**。`xlsx@0.18.5`、`jspdf@3.0.1`、`html2canvas@1.4.1`、`print-js@1.6.0` 均已在 `package.json`。

**挡路的死代码**（仅修挡路的，不做无关清理）

- `DatasetPanel.vue:3-35` `testDeviceApi` 写在 SFC 顶层块之外 → 不被编译，而模板 `:181` 的 `@click` 引用它 → **点"测试"必崩**
- `core/ChartComponents.ts` 整套（`chart-line/bar/pie/gauge/radar`）不在组件面板里，面板拖不出来，且只渲染占位 div → 与 `unified-chart` 重复的死实现
- `utils-project.ts:229-450` 调用 `scadaApi.uploadResource`，但 `api/scada/project/index.ts` 未导出该函数 → 运行时 TypeError
- `/scada` 无 `:id` 菜单入口（`router/modules/scada.ts:24`）能进但存不了

## 三、分批实施

### 批 A：运行态渲染器补全（组态与报表共同地基）

1. `runtime.vue` 弃用自写的 `renderComponent` stub，改为复用 `createComponentElement`：
   - `editorContainer` → 运行态容器 ref（DOM 内含 `.canvas-content`）
   - `setupComponentInteractions` → 传空函数（运行态不绑选中/拖拽/缩放手柄）
   - 其余 `create*Element` / `applyStyleToElement` / `svgManager` / `componentManager` 照常 import
2. 扩展 `applyDatasetValues`：按组件类型分派更新（SVG 走 `svgManager.updateLiquidLevel`/`updateSwitchState`，图表走 `updateChartData`，表格走行数据替换，文本/数值卡走文本），不再只写 `properties.text`
3. 验证：发布一个含 SVG 图元 + 图表 + 表格的组态，运行态能正确渲染并实时刷新

### 批 B：项目类型二元化

1. 前端新增 `api/scada/dashproject/index.ts`（对接已有的 `/DashProject/*`、`/DashProjectData/*`）
2. 项目列表页 `views/project/` **参数化**：接受 `kind: "scada" | "dash"`，按 kind 选择调用哪套 API —— **复用同一套页面组件，不复制两份**
3. 路由拆两个入口：`/project`（组态项目）、`/report/project`（报表项目）；编辑器 `/scada/editor/:id` 增加 `kind` query 决定读写哪套接口
4. `FuxaComponentPanel.vue` 增 `projectType` prop，在 `filteredGroups` computed（`:278`）里按**组件 name 白名单**过滤：报表只放 `table` / `unified-chart` / `text-card` / 新增的统计数值卡 / 基础文本与形状
5. 新造"统计数值卡"组件（大数字 + 单位 + 标题 + 环比）

### 批 C：报表运行态

1. 报表运行态 = 批 A 的运行态 + 报表外壳：顶部**查询条件栏**（时间范围 + 设备），改条件重新拉数（覆盖数据集的 `historyHours`）
2. 数据集走 `IotDatasetConfig` 的 `history` 模式（已存在），不新建实体
3. 导出：打印（print-js + 打印样式，隐藏工具栏）、Excel（xlsx，导出表格组件数据）、PDF（html2canvas + jsPDF 截图整页）

### 批 D：撤除 JimuReport

1. 删 `views/report/lowcode.vue`、`router/modules/report.ts` 中的低代码报表子路由、`platform-config.json` 的 `ReportUrl`、`config/index.ts` 的 `getReportUrl()`
2. **保留** `ReportDatasetController`（日用量 / 告警统计两个数据集端点）—— 转为报表的"平台内置数据集"源；`VerifyToken`（JimuReport 专用回调）随之删除
3. 部署文档 `docs/低代码报表JimuReport部署集成文档.md` 标注废弃

### 批 E：顺手修的挡路死代码

仅修**挡路**的：`DatasetPanel.vue` 的 `testDeviceApi`（必崩按钮）、`uploadResource` 缺失（运行时 TypeError）、`/scada` 无 id 入口。`ChartComponents.ts` 死实现在批 B 触及组件面板时一并删除。

## 四、风险

- **批 A 是主要不确定性**：`createComponentElement` 内部的 `create*Element` 分支若隐式依赖编辑器的响应式状态（选中项、缩放比例等），运行态复用会暴露出来，需逐个组件类型验证
- 组态发布后的运行态从未真正可用过，批 A 完成后可能暴露出后续的历史问题（如资源上传、图元资源路径）
